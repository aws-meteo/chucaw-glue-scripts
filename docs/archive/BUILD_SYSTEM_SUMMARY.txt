⚡ COMANDOS PRINCIPALES:

   .\test-glue-build.ps1

   # Primera compilación
   .\build-glue-deps.ps1

   # Rebuild (imagen cacheada)
   .\build-glue-deps.ps1 -SkipBuild

   # Clean build
   .\build-glue-deps.ps1 -CleanBuild

   # Upload a S3
   aws s3 cp build\glue-dependencies.zip s3://bucket/libs/

   # Crear Glue Job
   aws glue create-job --cli-input-json file://examples/glue-job-example.json

═══════════════════════════════════════════════════════════════════════════════

🎓 CONCEPTOS CLAVE:

   REPL Loop:
      Ciclo de verificación rápida (Read-Eval-Print-Loop) que te permite
      validar imports localmente en 2 min, no esperando 10+ min de Glue.

   Binary Compatibility:
      Los archivos .so (Linux) no son intercambiables con .pyd (Windows).
      Docker garantiza compilación en el mismo OS que AWS Glue.

   Fail Fast:
      Detectar errores ANTES de upload a S3. Ahorra tiempo y frustraciones.

   Optimization:
      Eliminar __pycache__, tests, docs reduce ZIP de ~80MB a ~45MB.

═══════════════════════════════════════════════════════════════════════════════

💡 TIPS:

   • Primera build toma 5-10 min (compila eccodes). Es normal.
   • Segunda build en adelante: ~30 seg (Docker cache).
   • Si cambias dependencies: Solo rebuild (no rebuild Docker image).
   • Usa -SkipBuild para rebuilds rápidos.
   • Verifica CloudWatch logs después de primer job en Glue.
